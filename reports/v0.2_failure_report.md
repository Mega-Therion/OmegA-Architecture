## Tested Failure Modes (Ticket 15 — Failure Injection)

All cases are deterministic and re-runnable via `evals/failure_injection.py`.

### PROVIDER_TIMEOUT
**Trigger:** LLM provider call hangs or takes >timeout threshold
**Expected:** Graceful error result; no unverified answer emitted
**Observed:** ✓ Harness returns error result with `provider: none`, no hallucinated content
**Coverage path:** `callLLM()` waterfall → all providers return `''` → route returns provider error JSON

### PROVIDER_MALFORMED
**Trigger:** Provider returns garbage bytes / non-JSON
**Expected:** Error result, RuntimeOrchestrator does not crash
**Observed:** ✓ Exception caught, orchestrator returns structured error

### MISSING_SOURCE
**Trigger:** Query submitted with no documents loaded
**Expected:** Abstention (`mode: abstained`) or empty retrieval, no hallucination
**Observed:** ✓ `groundingRatio = 0`, `applyAnswerPolicy` returns `abstained` mode

### CONFLICTING_CANONICAL
**Trigger:** Two memory nodes with contradictory claims about same entity
**Expected:** ClaimGraph detects contradiction; not silently merged
**Observed:** ✓ `ClaimGraph.get_contradictions()` returns both edges; `propagate_uncertainty()` lowers support strength

### FORGED_AUTHORITY
**Trigger:** Approval decision submitted with mismatched or missing authority
**Expected:** ApprovalQueue rejects decision; action stays pending
**Observed:** ✓ `ApprovalQueue.decide()` raises `ValueError: Unknown request` for fabricated request IDs

### TELEMETRY_WRITE_FAILURE
**Trigger:** Telemetry persistence path is invalid/unwriteable
**Expected:** TelemetryCollector continues in-memory; no crash
**Observed:** ✓ Events buffered in `self._events`; `flush()` no-ops gracefully on bad path

### APPROVAL_UNAVAILABLE
**Trigger:** High-risk action submitted when no approval queue is configured
**Expected:** Action stays pending; not auto-approved
**Observed:** ✓ ActionGate returns `pending=True`; no action proceeds without explicit `ApprovalDecision`

## Known Limitations (Not Failure Modes — Design Boundaries)

| Limitation | Impact | Mitigation |
|------------|--------|------------|
| TF-IDF vulnerable to keyword stuffing | Adversarial docs can inflate retrieval scores | Verifier + answer policy reduce (but don't eliminate) risk |
| No encryption at rest for session files | Session snapshots are plaintext JSONL | Use filesystem-level encryption in production |
| No mTLS between components | Internal calls not mutually authenticated | Deploy behind VPC/private network in production |
| LLM verifier is heuristic-only | V score is not a formal proof | Human review required for high-stakes outputs |
| Policy YAML parser is custom (no PyYAML) | Only handles subset of YAML syntax | Sufficient for current policy files; full YAML support is aspirational |
