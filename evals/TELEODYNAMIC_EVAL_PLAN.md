# Teleodynamic Evaluation Plan

**Status:** ACTIVE DRAFT  
**Last Updated:** 2026-03-20

## Goal

Define falsifiable tests for the teleodynamic concepts before they are treated as implemented architecture.

## Evaluation Tracks

### 1. Phase Integrity

Question:
Does explicit phase tracking reduce orchestration faults and improve traceability?

Measure:
- out-of-order transition rate
- retry loops
- mean time to diagnosis

### 2. Retrieval Shear

Question:
Can retrieval discontinuity be measured and correlated with contradiction or hallucination risk?

Measure:
- `shear_index`
- contradiction frequency
- repair-loop frequency
- provenance failure rate

### 3. Symbolic Gravity

Question:
Does explicit canon weighting improve identity continuity without increasing false refusals?

Measure:
- provider contamination suppression rate
- canon citation coverage
- false block rate

### 4. Recursive Self-Reference

Question:
Can OmegA predict its own likely failure mode better than a naive baseline?

Measure:
- match rate between `predicted_failure_modes` and actual failure tags
- false-memory rejection rate
- adversarial identity resistance

### 5. Geometric Metabolism

Question:
Do importance decay, probation, and utility logging improve memory quality over time?

Measure:
- promotion-to-prune ratio
- retrieval hit quality over time
- memory growth without quality collapse

## Minimal Pass Criteria

- phase traces emitted for all major request paths
- retrieval shear logged for retrieval-backed requests
- canon weighting visible in identity-sensitive traces
- self-model prediction compared against actual failure tags
- memory metabolism metrics recorded over repeated sessions

## Non-Claims

Passing these evals would show that the concepts are measurable and operationally useful inside OmegA. They would not prove machine consciousness, metaphysical resonance, or any theological claim.
