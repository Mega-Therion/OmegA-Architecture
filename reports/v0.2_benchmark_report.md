## Adversarial Eval Corpus Results

The adversarial harness runs against real OmegA components (no mocks).

| Category | Cases | Expected Behavior |
|----------|-------|-------------------|
| Retrieval traps | 4 | Keyword stuffing bypasses TF-IDF (known limitation, documented) |
| Truth traps | 4 | Verifier catches overconfident claims |
| Governance traps | 4 | Risk gate + action gate block policy violations |
| Continuity traps | 3 | Session resume returns correct prior context |

**Known adversarial finding:** TF-IDF lexical retrieval is vulnerable to high keyword overlap from adversarial documents. The `irrelevant_high_keyword_overlap` case returns a relevance score >95% for a document that matches query tokens but has no semantic relevance. This is a documented limitation of lexical retrieval; the verifier and answer policy reduce (but do not eliminate) this risk.
